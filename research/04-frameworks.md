# 04. Frameworks open source recommandes

Analyse des outils disponibles pour l'evaluation de LLM. Chaque framework est evalue selon sa compatibilite avec le contexte africain (langues, cout, reproductibilite).

| Framework | Role | Lien |
|-----------|------|------|
| **LM Evaluation Harness** | Recommande — standard de facto | [github.com/EleutherAI/lm-evaluation-harness](https://github.com/EleutherAI/lm-evaluation-harness) |
| **OpenCompass** | Multilingue — meilleur support langues non-anglaises | [github.com/open-compass/opencompass](https://github.com/open-compass/opencompass) |
| **HELM** | Methodologie — modele de reference scoring holistique | [github.com/stanford-crfm/helm](https://github.com/stanford-crfm/helm) |
| **DeepEval** | Industrialisation — pipeline automatise, metriques avancees | [github.com/confident-ai/deepeval](https://github.com/confident-ai/deepeval) |
| **Promptfoo** | CI/CD — tests unitaires de prompts et integration continue | [github.com/promptfoo/promptfoo](https://github.com/promptfoo/promptfoo) |
| **Langfuse** | Traçabilite — tracing et monitoring des evaluations | [github.com/langfuse/langfuse](https://github.com/langfuse/langfuse) |
| **Arize Phoenix** | Observabilite — debugging et visualisation des performances | [github.com/Arize-AI/phoenix](https://github.com/Arize-AI/phoenix) |
| **RAGAs** | RAG — evaluation des pipelines de Retrieval-Augmented Generation | [github.com/explodinggradients/ragas](https://github.com/explodinggradients/ragas) |
| **BIG-bench** | Reference — benchmark大规模协作 | [github.com/google/BIG-bench](https://github.com/google/BIG-bench) |
| **TruthfulQA** | Reference — evaluation de la veracite des modeles | [github.com/sylinrl/TruthfulQA](https://github.com/sylinrl/TruthfulQA) |
