# 04. Frameworks open source recommandés

Analyse des outils disponibles pour l'évaluation de LLM. Chaque framework est évalué selon sa compatibilité avec le contexte africain (langues, coût, reproductibilité).

| Framework | Rôle | Lien |
|-----------|------|------|
| **LM Evaluation Harness** | Recommandé — standard de facto | [github.com/EleutherAI/lm-evaluation-harness](https://github.com/EleutherAI/lm-evaluation-harness) |
| **OpenCompass** | Multilingue — meilleur support langues non-anglaises | [github.com/open-compass/opencompass](https://github.com/open-compass/opencompass) |
| **HELM** | Méthodologie — modèle de référence scoring holistique | [github.com/stanford-crfm/helm](https://github.com/stanford-crfm/helm) |
| **DeepEval** | Industrialisation — pipeline automatisé, métriques avancées | [github.com/confident-ai/deepeval](https://github.com/confident-ai/deepeval) |
| **Promptfoo** | CI/CD — tests unitaires de prompts et intégration continue | [github.com/promptfoo/promptfoo](https://github.com/promptfoo/promptfoo) |
| **Langfuse** | Traçabilité — tracing et monitoring des évaluations | [github.com/langfuse/langfuse](https://github.com/langfuse/langfuse) |
| **Arize Phoenix** | Observabilité — debugging et visualisation des performances | [github.com/Arize-AI/phoenix](https://github.com/Arize-AI/phoenix) |
| **RAGAs** | RAG — évaluation des pipelines de Retrieval-Augmented Generation | [github.com/explodinggradients/ragas](https://github.com/explodinggradients/ragas) |
| **BIG-bench** | Référence — benchmark à grande échelle | [github.com/google/BIG-bench](https://github.com/google/BIG-bench) |
| **TruthfulQA** | Référence — évaluation de la véracité des modèles | [github.com/sylinrl/TruthfulQA](https://github.com/sylinrl/TruthfulQA) |
